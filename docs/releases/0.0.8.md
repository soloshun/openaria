# Lumis SDK 0.0.8

Lumis SDK 0.0.8 combines the Sprint 4 policy/proposal milestone and Sprint 5
verification/learning milestone while preserving a non-executing core.

## Highlights

- Strict versioned playbook, default-deny policy, action-proposal, and approval-decision contracts.
- Diagnosis/evidence digests, pinned document revisions, proposal expiry, typed bounded parameters,
  canonical JSON, and stable SHA-256 hashes.
- Deterministic proposal evaluation where unknown actions fail closed and high risk never
  auto-approves.
- Explicit verification requests, checks, passed/failed/unknown/timed-out outcomes, and mandatory
  escalation for every non-passing result.
- Persistent rejected and superseded truth transitions in the SQLite reference adapter and
  PostgreSQL plugin.
- Reusable-only memory retrieval with visible lexical, filter, and truth score components.
- Deterministic synthetic replay metrics for diagnosis, escalation, and verification regression.
- Collision-safe discovery for multiple installed plugin manifests.

## Compatibility

The original lifecycle models remain available. The public `MemoryStore` protocol now requires
`record_truth_transition`; third-party adapters should implement the 0.0.8 contract before
declaring compatibility.

The corrected official plugins are independently versioned at `0.1.1`:

```bash
pip install \
  "lumis-sdk==0.0.8" \
  "lumis-sdk-postgres-memory==0.1.1" \
  "lumis-sdk-http-json-evidence==0.1.1"
```

Plugin installation grants no authority by itself. The PostgreSQL plugin requires explicit network
and secret authority at composition; the HTTP evidence plugin requires explicit network and secret
authority. Neither provides action execution.

## Install

```bash
uv add "lumis-sdk==0.0.8"
```

or:

```bash
pip install "lumis-sdk==0.0.8"
```
