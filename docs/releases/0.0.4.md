# Lumis SDK 0.0.4

Lumis SDK 0.0.4 fixes the structured-rule evidence distinction reported in issue #19.

## Fixed

- `spec.evidence.required` remains a hard precondition that prevents a rule from matching.
- `spec.diagnosis.missingEvidence` now records evidence that would improve, contradict, or
  confirm an already matched diagnosis.
- Matched diagnoses carry those values into `DiagnosisResult.missing_evidence` and triage
  missing context.
- Rule validation rejects the same value appearing in both required and outstanding lists.
- `lumis rules test` exposes the selected diagnosis and its outstanding evidence.

The new field is optional, so existing rules retain their current matching behavior.

## Install

```bash
uv add "lumis-sdk==0.0.4"
```

or:

```bash
pip install "lumis-sdk==0.0.4"
```
