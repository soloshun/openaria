# Lumis SDK 0.0.2

Lumis SDK 0.0.2 adds the first portable structured Diagnosis-as-Code contract and the
open-source repository safety baseline.

## Highlights

- Strict `lumis.dev/v1alpha1` single-file `DiagnosisRule` documents.
- Compound `all`, `any`, and `not` conditions over structured fields.
- Exact, contains, prefix, regex, and numeric comparisons.
- Required-evidence checks and retained evidence references.
- Deterministic priority, specificity, and stable-order ranking.
- Candidate explanations with matched and failed conditions.
- `lumis rules test` for bounded JSON fixtures.
- Generated JSON Schema and migration guidance from `all_contains`.
- CI on Python 3.11, 3.12, and 3.13.
- Locked dependency audit, Bandit SAST, Dependabot, CODEOWNERS, and contribution templates.

## Compatibility

The existing `DeterministicRule` and `diagnose_text_with_explanation` APIs remain available.
Projects should migrate a complete rule collection together because mixed legacy and structured
collections are rejected to avoid ambiguous cross-engine ordering.

Structured evaluation remains local, deterministic, model-free, and non-executing.

## Install

```bash
uv add "lumis-sdk==0.0.2"
```

or:

```bash
pip install "lumis-sdk==0.0.2"
```
